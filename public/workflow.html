✦ Spark Workflow

🚀 Developer Workflow Guide

How to develop, test, and deploy Spark from start to finish.

📋 Overview

Spark is deployed on Vercel with a Supabase cloud backend. The code lives on GitHub. You develop locally, push to GitHub, and Vercel auto-deploys.

💻 Local Code → 🐙 GitHub Push → ▲ Vercel Auto-Deploy → 🌐 Live Site

📥 Starting a New Session

1

Open Command Prompt

Press Win + R, type cmd, press Enter.

2

Navigate to project folder

cd C:\Users\andre\Documents\Deepseek\Code App
3

Pull the latest code from GitHub

This ensures your local code matches what's deployed on Vercel.

git pull origin main
If you get an error about uncommitted changes, run git stash first, then pull.
4

Start Reasonix

npx reasonix@latest code

Then /preset max for Pro model, /stats to check.

💻 Making Changes

1

Tell Reasonix what to change

Describe the feature or fix you want. Reasonix will read the relevant files, propose edits, and apply them.

2

Test locally (optional)

If you want to preview before deploying, start the local server:

node server.js

Then open http://localhost:3000. This connects to your cloud Supabase database, so data is shared with the live site.

Press Ctrl + C in the terminal to stop the server when done testing.

🚀 Deploying Changes

1

Commit and push to GitHub

Tell Reasonix to push the changes. It will run:

git add -A git commit -m "description of changes" git push origin main
2

Vercel auto-deploys

Within 1-2 minutes, Vercel detects the push, builds the project, and deploys it. Your live site updates automatically.

3

Verify the live site

Check your changes at https://spark-social-app-ys49.vercel.app

🗄️ Database Notes

⚠️ Common Pitfalls

  • Don't edit .env values without also updating Vercel's Environment Variables
  • Always git pull first when starting a new session - your local code might be behind
  • Supabase keys are sensitive - never commit them to GitHub
  • Socket.io won't work on Vercel - chat uses Supabase Realtime instead
  • Photo uploads now go to Supabase Storage, not local uploads/ folder
