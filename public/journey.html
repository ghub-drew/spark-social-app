My Journey: Building Spark with Vibe Coding

How a social app was built from scratch using AI - no prior coding experience needed

What is Vibe Coding? A term for building software by describing what you want in plain English to an AI, letting it write the code, and iterating until it feels right. You don't need to know programming languages - just your ideas.

The Beginning

Spark started as an idea: a simple social app where people could connect, chat, and build friendships. The goal was to create something real - not just a concept - using AI as the developer.

Step 1
Get an AI Coding Assistant - Signed up for Reasonix (a vibe coding platform) and got an API key from DeepSeek.
Step 2
First Prompt - Told the AI: "Create a social app with user registration, profiles, chat, and admin panel." The AI generated the entire project structure.
Step 3
Iterate & Refine - Made dozens of small changes: changed the theme from pink to green, renamed "Messages" to "Chat", added error logging, improved the UI.
Step 4
Add Features - Contact form, FAQ page, Terms & Privacy, documentation, changelog, workflow guide.
Step 5
Database Upgrade - Migrated from local JSON file (lowdb) to Supabase PostgreSQL for real scalability.
Step 6
Go Live - Deployed to Vercel, connected a custom domain (cebuspark.com), and the app was live to the world.

The Complete Workflow

Here's how every change goes from idea to live website:

1

Get an AI Coding Tool

Sign up for a vibe coding platform. I used Reasonix which runs on top of DeepSeek AI. You can also use Cursor, Claude, or similar tools.

Go to platform.deepseek.com -> Sign up -> Get API key Install: npx reasonix@latest code
2

Start Your Project

Describe your app in plain English. Tell the AI what you want to build. It will create the files and structure for you.

"Create a social app with login, profile pages, real-time chat, member browsing, and an admin panel."
3

Test Locally

The AI starts a local server so you can see your app in the browser immediately.

Open http://localhost:3000 -> See your app running
Changes appear instantly on localhost. Nothing goes live until you say so.
4

Iterate & Improve

Keep telling the AI what to change. Change colors, add features, fix issues - all through conversation.

"Change the theme to green" "Add a contact form" "Improve the mobile layout" "Fix the profile photo upload"
5

Set Up Database (Optional but Recommended)

For a production app, use a real database. Supabase offers a generous free tier with PostgreSQL.

1. Create account at supabase.com 2. Create a new project (free) 3. Let the AI connect it to your app 4. Migrate your data
6

Push to GitHub

Initialize git and push your code to GitHub. This is your source of truth and backup.

git init git add -A git commit -m "Initial commit" git remote add origin https://github.com/your-username/your-repo.git git push -u origin main
7

Deploy to Vercel

Connect your GitHub repo to Vercel for automatic deployments on every push.

1. Go to vercel.com 2. Import your GitHub repo 3. Add environment variables 4. Click Deploy -> Live!
8

Custom Domain (Optional)

Connect your own domain name for a professional touch.

1. Buy a domain (e.g., from Namecheap, GoDaddy) 2. Add it in Vercel Dashboard -> Domains 3. Update DNS settings 4. Your app is live at your custom domain!
9

Keep Improving

Every time you want to make changes:

1. git pull origin main (get latest code) 2. npx reasonix@latest code (start AI) 3. Describe your changes 4. Test on localhost 5. Say "push it" -> Vercel auto-deploys

Key Lessons Learned

  • Start simple - Get the core working first, then add features
  • Test locally - Preview everything on localhost before pushing live
  • Use a real database early - JSON files don't scale (we learned this the hard way!)
  • Document as you go - Changelog, workflow guide, and docs saved us many times
  • Git push = deploy - The workflow is just: edit, test, push, done
  • Vibe coding works - Describing what you want in plain English to AI is a real way to build software
Final thought: You don't need to be a programmer to build a web app anymore. With AI, you just need a clear idea and the willingness to iterate. Spark went from a concept to a live app at cebuspark.com in just a few days of conversation.
