How I Built Spark

I didn't use just one AI - I used three.

Claude helped architect the foundation and structure of Spark.

DeepSeek handled the heavier coding workflows with its long context capabilities.

ChatGPT helped with visuals, branding prompts, creative direction, and UI inspiration.

Each tool brought something different to the process, and Spark is better because of it.

The Beginning

Spark started as an idea: a simple social app where people could connect, chat, and build friendships. I didn't know how to code, but I knew AI could help me build it. So I jumped in.

Step 1
Claude - The Architect I started with Claude because it's great at planning. It helped me think through the app structure, the tech stack, and how everything would fit together before writing a single line of code.
Step 2
DeepSeek - The Builder Once the plan was solid, I switched to DeepSeek through Reasonix. Its long context let me work on bigger pieces without losing track. The whole backend, database logic, and API routes were built here.
Step 3
ChatGPT - The Creative For the look and feel, I turned to ChatGPT. It helped with color palettes, icon ideas, branding prompts, and the overall vibe you see on the site today.
Step 4
Iterate Across All Three I bounced between them - Claude for refactoring ideas, DeepSeek for the actual code changes, ChatGPT for quick design tweaks. Each had its strength.
Step 5
Database & Infrastructure Supabase replaced the local JSON database. Vercel handled hosting. All configured through conversation with AI.
Step 6
Go Live Deployed to Vercel, connected cebuspark.com, and Spark was live to the world. Built without writing a single line of code myself.

The Complete Workflow

Here's how every change goes from idea to live website:

1

Choose Your AI Toolkit

I used Claude for planning, DeepSeek + Reasonix for coding, and ChatGPT for design. Pick what works for you - each has different strengths.

Claude - Planning & Architecture
DeepSeek + Reasonix - Long context coding
ChatGPT - Visuals & Branding
2

Start Your Project

Describe your app in plain English. The AI creates all the files.

"Create a social app with login, profiles, chat, and admin panel."
3

Test Locally

See your app at http://localhost:3000. Changes appear instantly. Nothing goes live until you say so.

4

Iterate & Improve

Keep telling the AI what to change - colors, features, fixes - all through conversation.

"Change the theme to green"
"Add a contact form"
"Improve mobile layout"
5

Set Up Database

Supabase offers a generous free tier with PostgreSQL. Let the AI connect it to your app.

6

Push to GitHub

git init && git add -A && git commit -m "Initial commit"
git push -u origin main
7

Deploy to Vercel

Connect your GitHub repo to Vercel. Every push auto-deploys.

8

Custom Domain

Connect your own domain. Spark lives at cebuspark.com.

9

Keep Improving

Pull latest code, start Reasonix, describe changes, test, push - repeat.

git pull origin main
npx reasonix@latest code

Key Lessons Learned

  • Start simple - Get the core working first, then add features.
  • Test locally - Preview everything on localhost before pushing live.
  • Use a real database early - JSON files don't scale (we learned this the hard way!).
  • Document as you go - Changelog, workflow guide, and docs saved us many times.
  • Git push = deploy - The workflow is just: edit, test, push, done.
  • Use different AIs for different jobs - Claude for architecture, DeepSeek for coding, ChatGPT for design. Each has strengths. Play to them.
  • Vibe coding works - Describing what you want in plain English to AI is a real way to build software. You just need an idea and the patience to iterate.
Final Thought
You don't need to be a programmer to build a web app anymore. With AI, you just need a clear idea and the willingness to iterate. Spark went from a concept to a live app at cebuspark.com in just a few days of conversation.
